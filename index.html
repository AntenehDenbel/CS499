About This Portfolio

Overview

This ePortfolio is the capstone of my Bachelor of Science in Computer Science at Southern New Hampshire University. It documents three substantial enhancements to a single artifact, the Warehouse Inventory Management Android application I originally produced in CS 360.

Each enhancement targets one of the three core competency areas of the program: software design and engineering, algorithms and data structures, and databases with a security focus. Reading them in order shows the same project evolve from a monolithic Activity into a tested, indexed, and server-mediated application.

By that point I had already shipped two products as a solo developer for local businesses, including a web application and a mobile app that are still in active use. The Original Artifact

Starting Point
CS 360 Android Java + Kotlin SQLite (Room)

Warehouse Inventory Management App

An Android application written primarily in Java, with Kotlin used for Jetpack Compose theming. The app uses Room (SQLite) for local persistence, Material Components for the user interface, a RecyclerView grid of inventory cards, and conditional SMS notifications for low-stock alerts.

In its original form the application stored user passwords in plaintext, executed a literal WHERE username = :username AND password = :password query for authentication, performed every read as a linear scan over the full inventory table, and provided no search functionality. The three enhancements below were designed in response to those specific weaknesses.

Enhancement 1 Software Design & Engineering Outcomes 1, 2, 4

MVVM Refactor with Repository Pattern

Restructures the application around the Model-View-ViewModel pattern with a Repository interface mediating database access. The persistent delete bug is corrected, the allowMainThreadQueries() anti-pattern is removed, business logic is migrated from Java to Kotlin, raw new Thread() calls are replaced with a single-thread executor, and unit tests are introduced where none previously existed. This enhancement establishes the structural foundation that makes the next two possible without rewriting the user interface.

Enhancement 2 Algorithms & Data Structures Outcomes 3, 4

Indexed Read Path with Trie, HashMap, and Min-Heap

Introduces an ItemIndex component that maintains three coordinated structures: a hand-implemented Trie supporting prefix search in O(k + m), a HashMap providing O(1) lookup by primary key, and a hand-implemented binary min-heap with a position map providing O(log n) updates. The accompanying narrative includes a Big-O complexity table and benchmarked wall-clock measurements at three input sizes (100, 1,000, and 10,000 items), with a randomized stress test that verifies equivalence before timing.

Enhancement 3 Databases & Security Outcomes 3, 4, 5

Server-Mediated Architecture with PBKDF2, RBAC, and Audit Log

Rebuilds the data and security layers around a server-mediated architecture. Plaintext password storage is replaced with PBKDF2-HMAC-SHA-256 hashing over per-user 16-byte salts at 120,000 iterations. A flat permissions model is replaced with role-based access control enforced server-side. The absence of an audit trail is corrected with an append-only log defended by multiple layers, including a reflection-based build-time test that fails the build if the audit DAO ever gains an Update or Delete method.
